Basic Content

1. What is version control and how does Git work?

Version control is a system that helps track changes to files over time. It allows multiple developers (or even a single developer) to collaborate, manage different versions of their code, and track history in a way that enables easy rollback or recovery of earlier versions. It also helps in merging changes made by different contributors to the same project.

Git is a distributed version control system. It allows each user to have a complete local copy of the repository, with the entire history of changes. This means Git works even when you're offline, and you can sync your local changes with a remote repository when you are ready.

2. What are the 3 possible states of files? (Modified, Staged and Committed)

Modified: The file has been changed in the working directory (your local machine), but the changes have not yet been staged (added to the index) or committed to the repository.

Staged: The file is in the staging area (index), meaning it is prepared and ready to be committed to the local repository. It has been added using the git add command.

Commited: The changes are saved to the local repository, which means they have been stored and the history has been updated. A committed file is part of the version history.

3. How to start a git repository in a project?

To start a Git repository in a project, follow these steps:

  1. Navigate to your project folder using the command line or terminal.
  2. Initialize a Git repository by running: git init

This command creates a hidden .git directory, marking the folder as a Git repository.

4. How to store changes to files in Git?

To store changes to files in Git, you follow a few steps:

  1. Stage the files using git add: git add 'filename' or git add .
  2. Commit the changes using git commit: git commit -m 'feat(init): Create initial file'

The git add command stages the changes, and git commit stores them in the repository's history.

5. How to view the change history?

To view the change history in Git, you can use the git log command: git log

This will show a list of commits, including the commit ID, author, date, and commit message. You can add additional flags to customize the output, for example: git log --oneline

6. How to add files to an already Committed file, remove a file from the Staged state and discard changes to a Modified file? (amend, reset, checkout)

(Amend) If you want to modify the most recent commit (for example, adding a new file or changing the commit message):

  1. Stage the new file: git add 'filename'
  2. Amend the last commit: git commit --amend

This will open your default text editor where you can modify the commit message, or you can add the --no-edit flag to keep the existing message.

(Reset) If you've staged a file but decide not to commit it, you can remove it from the staging area: git reset 'filename'

This moves the file from the staged area back to the modified state, without losing any changes made to the file.

(Checkout) If you want to discard local changes and restore the file to its last committed state, use: git checkout -- 'filename'

This command replaces the file with its version from the last commit, effectively discarding your local changes.
